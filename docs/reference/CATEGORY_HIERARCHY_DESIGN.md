# 分类层级设计

> 状态：已实现。代码已接入生产调用链对应的前后端路径，并完成单元、类型、构建和隔离无头 Chrome 模拟数据验证；尚未部署，因此生产截图与生产域名回归需在用户明确要求部署后执行。

## 背景与目标

当前分类是严格的一层结构：书签通过 `category_id` 直接归属分类，分类使用全局 `sort` 排序，首页导航与内容分区一一对应。

目标是在保持现有书签归属模型和旧数据兼容的前提下，支持一级分类与二级分类，并为未来扩展保留稳定的数据边界。

首版目标：

- 最多支持两层分类。
- 一级分类和二级分类都可以直接包含书签。
- 一级分类可以作为内容节点，同时包含二级分类。
- 旧数据、旧备份和现有一层分类行为继续有效。

## 已确认需求

### 层级与内容

- 数据模型使用可空的父分类标识表达层级。
- 产品界面最多允许两层，不允许创建三级分类。
- 父分类可以保存直属书签。
- 二级分类可以保存书签，但不能继续包含子分类。

### 导航交互

- 点击父分类标题跳转到父分类内容。
- 展开二级菜单使用独立箭头或展开按钮，不占用标题点击行为。
- 各导航模式默认折叠二级分类，只有独立箭头负责改变菜单展开状态。
- 左侧导航、顶部导航和移动端抽屉必须提供一致的层级语义。

### 后台管理

- 普通列表按一级分类分页，每页默认只展示一级分类；二级分类由父级箭头展开。
- 排序只允许在同一父级内进行。
- 拖拽不改变父级；跨父级移动通过分类编辑表单的“上级分类”字段完成。
- 删除含子分类的父分类时阻止操作，要求先移动或删除子分类；批量删除同样不能绕过该限制（删除规则详见“删除规则”）。

### 导入兼容

- 旧 CF-Navs 备份缺少父级字段时，所有分类视为一级分类。
- 新备份保存父级字段并恢复原有层级。
- Sun-Panel 分类保持为一级分类，其 `children` 继续按书签解析。
- 浏览器书签 HTML 的前两层有效文件夹映射为一级和二级分类。
- 超过两层的剩余路径合并进二级分类标题，例如 `开发 / 前端 / 框架` 映射为一级“开发”、二级“前端 / 框架”。
- 深层路径压平后，`开发 / 前端`（直接在“前端”下的书签）与 `开发 / 前端 / 框架` 会成为“开发”下两个**平级且独立**的二级分类，二者不再保留从属关系。
- 合并进二级标题的分隔符必须与后台路径展示分隔符（见“书签编辑”）区分，避免二级标题 `前端 / 框架` 与真实两级路径 `开发 / 前端` 在界面上都呈现为“看似三级”的字符串，误导用户和按路径匹配的合并导入。建议压平时使用非展示分隔符（如 `›`），或在展示层对子分类标题内已有的 ` / ` 做转义。

## 数据与接口设计

### 分类契约

为分类增加：

```ts
parent_id: number | null
```

`Category`、`PublicCategory` 和分类新增/更新请求使用相同字段。书签继续只保存一个 `category_id`，不引入多分类归属。

### 数据库

- `categories` 增加可空 `parent_id`。
- 现有分类迁移后统一为 `parent_id = NULL`。
- 增加 `(parent_id, sort, id)` 索引。
- `sort` 表示同一父级内的顺序，不再表示所有分类的单一全局顺序。
- 迁移必须幂等，并兼容新安装和已有 D1 数据库。SQLite 的 `ALTER TABLE ... ADD COLUMN` 本身不幂等（重复执行会因列已存在而报错），因此幂等需通过先查 `PRAGMA table_info(categories)` 判断 `parent_id` 是否存在、再决定是否加列来实现，不能仅依赖重复执行 `ADD COLUMN`。
- 不依赖 D1 外键级联保证业务规则，父级校验和删除保护由数据访问层显式执行。

### 写入校验

新增或修改分类时必须验证：

- `parent_id` 为空，或指向已存在的一级分类。
- 分类不能以自身作为父级。
- 二级分类不能成为其他分类的父级。
- 已经拥有子分类的一级分类不能移动到另一个父分类下。
- 不允许通过更新形成三级结构。循环引用检查在“父级必须指向一级分类”约束下属于冗余防御：两层结构无法成环，保留该检查仅作保险，不代表存在真实成环路径。
- 创建或移动分类时，初始 `sort` 必须在目标父级作用域内计算（同一 `parent_id` 内取 `MAX(sort) + 1`），不再使用跨父级的全局最大值，否则会破坏“每个父级内 `sort` 连续”的假设。

### 排序接口

分类排序请求调整为：

```ts
type CategorySortReq = {
  parent_id: number | null
  ids: number[]
}
```

服务端验证所有 ID 唯一、存在且属于指定父级，只更新该同级集合的顺序。书签排序接口保持不变。

`ids` 必须是指定父级下的**全部**兄弟分类，服务端按下标写 `sort`。不接受部分子集重排：若只传部分 ID，写入的 `sort` 会与未传兄弟的现有值冲突或错乱。校验需同时确认传入 ID 集合与该父级现有子分类集合完全一致。

### 删除规则

- 分类存在子分类时返回明确的冲突错误，不删除任何数据。
- 无子分类的分类删除后，继续删除其直属书签，保持现有行为。
- 批量删除中只要包含仍有子分类的分类，整个请求失败，不进行部分删除。
- 前端确认信息显示直属书签数量和阻止删除的子分类数量。

### 查询与缓存

- 分类接口继续返回扁平数组，由纯逻辑层构建树形结构。
- 公开聚合与后台聚合响应增加 `parent_id`，其他数据形状不变。
- 分类层级变化继续更新 `data_version`，并失效公开与后台相关缓存。
- 旧浏览器快照缺少 `parent_id` 时，在缓存读取边界规范化为 `null`，不要求用户清除站点数据。

## 首页与导航设计

### 内容结构

- 首页按一级分类的同级顺序渲染分组。
- 一级和二级分类内容默认折叠；展开一级分类后先显示直属书签和二级分类标题，再由二级分类箭头展示对应书签。
- 一级分类没有直属书签时，只保留可定位的分组标题，不渲染空书签卡片区。
- 二级分类使用独立且稳定的内容锚点，供导航和滚动高亮使用。

### 导航表现

- 左侧导航使用缩进的二级列表。
- 顶部导航由父分类旁的独立按钮打开锚定子菜单。
- 移动端左侧抽屉使用可折叠子菜单。
- 子菜单支持键盘焦点、`Escape`、外部点击关闭和导航完成后关闭。
- 父分类数量表示直属书签数，不混入后代书签，避免点击后的内容与数量不一致。

### 搜索与滚动

- 搜索索引加入完整分类路径。
- 搜索父分类名称时可以匹配其后代分类中的书签。
- 搜索结果只显示存在匹配内容的父级和子级结构，并展开匹配路径以直接呈现结果。
- 明确导航到二级分类时，首页内容区展开目标所需路径；导航菜单本身仍由箭头控制。
- 已知取舍：搜索父分类名可命中其后代书签（本节），但导航角标只算直属书签、点击父分类只进直属内容（见“导航表现”）。二者组合会造成“搜到的”与“点进去看到的”不一致。这是为保持角标与点击行为一致而接受的取舍；对仅通过后代命中的父级，搜索结果应给出可下钻到对应子分类的明确入口，避免用户误以为内容丢失。

## 后台界面设计

### 分类列表

- 普通模式每页显示 10 个一级分类，子分类默认折叠并由父级箭头展开。
- 搜索命中二级分类时保留其父级并展开匹配分支；命中父级时显示并展开该父级及全部子级。
- 父分类行分别展示直属书签数和子分类数。
- 二级分类使用缩进、连接线或层级图标表达归属，不使用嵌套卡片。

### 排序与移动

- 一级分类排序显示完整一级分类列表。
- 子分类只在所属父分类内排序。
- 不提供自由跨层拖拽，避免排序和改变父级混为同一操作。
- 编辑分类时提供“上级分类”选项：无上级表示一级分类，候选项只列出合法一级分类。

### 书签编辑

- 分类选择器按层级显示 `父分类 / 子分类`。
- 分类选择器中的一级分类默认折叠，使用独立箭头展开二级分类。
- 一级分类和二级分类均可被选中。
- 后台书签列表的分类列、搜索和排序使用完整分类路径，避免同名子分类难以区分。

## 导入、导出与合并

- 新版 JSON 导出保存每个分类的 `parent_id`。
- 导入规范化将缺失或无效的旧字段转换为 `null`。
- 合并导入按规范化分类路径匹配分类，不再只按标题匹配。
- 不同父分类下允许存在同名子分类。
- 导入前完成深度、父级存在性和书签分类引用校验，失败时不执行部分写入。
- 替换导入保持当前事务式清空和重建语义，写入顺序必须先一级分类、再二级分类、最后书签。
- 无论 replace 还是 merge，写入二级分类前必须完成老一级分类 ID → 新一级分类 ID 的映射，并用该映射重写二级分类的 `parent_id`；书签的 `category_id` 同样按分类 ID 映射重写。ID 重映射不仅覆盖书签引用，也必须覆盖 `parent_id`。

## 模块职责

- 共享类型定义稳定的数据契约。
- 纯分类树模块负责建树、路径、深度、同级排序、分页和搜索祖先保留。
- 数据访问层负责迁移、父级校验、同级排序和删除保护。
- 路由层负责请求解析与错误响应，不包含树形展示规则。
- 首页和后台视图只消费树形派生结果，不直接推导数据库关系。
- 导入模块负责外部层级到两层分类契约的转换。

## 非目标

首版不包含：

- 任意深度分类树。
- 一个书签归属多个分类。
- 拖拽时自动改变父级。
- 删除父分类时自动提升子分类。
- 删除父分类时级联删除整棵分类树。
- 父分类自动聚合并展示全部后代书签。
- 未经实际性能测量提前引入虚拟列表或新的全局状态框架。
