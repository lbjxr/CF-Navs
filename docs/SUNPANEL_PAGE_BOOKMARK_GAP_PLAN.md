# Sun-Panel 页面设置与书签管理对照清单

本文档基于本仓库内 `sun-panel/` 原项目源码，对照 CF-Navs 当前实现，聚焦页面设置和书签管理两块。CF-Navs 是 Cloudflare Workers + D1 的轻量单用户导航面板，因此这里按“轻量版本应补齐”和“原版大型能力暂缓”分层。

## 原版功能摘录

### 页面设置

- LOGO 文案：`logoText`，首页主标题可配置。
- 时钟模块：秒显示开关。
- 搜索栏：是否显示搜索框、是否启用站内/图标搜索项。
- 系统监控：是否显示、是否显示标题、公开访问模式是否显示。
- 图标样式：详情图标 / 小图标、隐藏描述、隐藏标题、文字颜色。
- 壁纸：上传图片、自定义图片地址、背景模糊、遮罩。
- 内容区：最大宽度、单位 `px/%`、左右边距、顶部边距、底部边距。
- 自定义页脚：`footerHtml`。
- 重置配置和保存配置。

### 书签管理

- 分组管理：新增、编辑、删除、拖拽排序。
- 书签管理：新增、编辑、删除、按分组加载、拖拽排序。
- 书签字段：标题、图标、URL、描述、打开方式。
- 打开方式：当前页、新窗口、当前页弹层。
- 图标编辑：文字图标、图片 URL/上传、图标背景色。
- 自动获取 favicon。
- 批量添加书签。
- Sun-Panel JSON 导入/导出。

## 当前已实现

- 分类/分组：创建、编辑、删除、拖拽排序。
- 书签：创建、编辑、删除、搜索过滤、拖拽排序、前台右键编辑。
- 基础设置：站点标题、标题颜色、标题字号、公开模式、主题、图床地址。
- 背景：纯色、渐变、图片 URL、模糊、遮罩透明度、遮罩颜色。
- 搜索引擎：默认引擎、引擎列表、新增/删除、查询模板校验。
- 卡片：详情/极简样式、尺寸、图标大小、详情描述开关、背景色/透明度。
- 图标：服务端 favicon、favicon.im、Google s2、本地文字 SVG、图标缓存回退。
- 数据：CF-Navs 备份导入/导出、SunPanel 导出导入转换。

## 本轮已补齐

- 首页搜索框显示开关：`search_box_show`。
- 搜索引擎选择器显示开关：`search_engine_selector_show`。
- 内容区布局设置：`content_layout.max_width`、`max_width_unit`、`margin_x`、`margin_top`、`margin_bottom`。
- 极简卡片标题显示开关：`card_icon_show_title`。
- 卡片文字颜色设置：`card_text_color`，留空时跟随主题。
- 自定义页脚 HTML：`footer_html`。
- 新字段已进入共享类型、默认设置、公开接口、设置接口校验、初始化 SQL 和首页渲染链路。
- 当前页弹层打开：`open_method=3`，使用 iframe 弹层并提供新窗口打开兜底。
- 单书签图标背景色：`icon_background_color`。
- SunPanel 导入保留 `openMethod=3` 和图标背景色。

## 仍未实现的细节

- 上传能力：原版后端提供文件上传；当前只支持跳转外部图床。
- 批量添加书签：当前只有导入备份，没有后台批量创建表单。
- 配置重置按钮：当前可手动改回默认值，但没有一键重置。
- 自定义 CSS / JS：类型和后端已有字段，但设置面板尚未暴露，首页也未执行自定义 JS。
- 时钟、系统监控、多用户、窗口应用：属于原版桌面面板能力，和 CF-Navs 轻量定位差异较大，建议不作为页面设置/书签管理第一阶段范围。

## 开发计划

### 阶段 1：页面设置补齐（已完成）

- 扩展设置数据结构并保持老库兼容。
- 补齐搜索框显示、引擎选择器、内容区布局、极简标题、文字颜色、页脚。
- 接入首页渲染和设置面板保存。
- 通过 `npm run type-check` 验证。

### 阶段 2：书签模型增强（部分完成）

- 已为 `bookmarks` 增加 `icon_background_color` 兼容字段。
- 已设置 `open_method=3` 的类型、API 校验和前台弹层视图。
- 已在后台编辑弹窗补充图标背景色。
- 已在 SunPanel 导入时保留弹层打开方式和背景色。

### 阶段 3：效率工具

- 后台增加批量添加书签入口，支持按行解析 `标题 URL 描述 分类`。
- 增加一键重置页面设置，重置前二次确认。
- 补充导入预览，显示将导入/覆盖的分类、书签、设置数量。

### 阶段 4：高级样式

- 暴露自定义 CSS，并在首页安全注入。
- 自定义 JS 建议暂缓或仅管理员可见预览后启用，因为公开页面执行任意脚本会带来明显风险。
- 根据实际需求决定是否引入时钟模块；系统监控不建议在 Cloudflare Workers 版本实现。
